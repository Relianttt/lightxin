# 首页redesign 设计说明文档

> 版本 v0.1 · 2026-04-18  
> 基于产品讨论整理，供开发与后续设计迭代使用

---

## 一、设计原则

首页的核心定位是**最简单的信息展示**，不做功能入口的堆砌。用户大部分情况下是有明确目的才打开应用，首页的职责是在恰当的时机展示恰当的一条信息，其余时间保持安静。

- 有任务时：展示一张智慧提醒卡片
- 无任务时：首页回归干净状态，问候语作为唯一主体
- 任何时候不在首页堆放多张静态信息卡片

---

## 二、页面结构

### 2.1 无提醒状态

```
┌─────────────────────────────┐
│                             │
│                             │
│                             │
│      下午好，何宇翔          │  ← 大标题（居中，放大）
│      今天没课，下午自由安排   │  ← 副标题（居中，放大，可动态切换）
│                             │
│                             │
│                             │
│  ──────────────────────── 底栏 ──  │
└─────────────────────────────┘
```

- 问候语整体**垂直居中**于去除底栏后的可用区域
- 大标题与副标题间距相较有提醒状态时**适当放大**
- 页面其余区域留白，不填充任何内容

### 2.2 有提醒状态

```
┌─────────────────────────────┐
│  下午好，何宇翔              │  ← 大标题（左对齐，正常尺寸）
│  今天没课，下午自由安排       │  ← 副标题（左对齐，正常尺寸）
│  ─────────────────────────  │  ← 分割线
│ ┌─────────────────────────┐ │
│ │  智慧提醒卡片            │ │
│ └─────────────────────────┘ │
│                             │
│  ──────────────────────── 底栏 ──  │
└─────────────────────────────┘
```

- 问候语整体**左对齐**，位于页面顶部
- 分割线位于问候语与卡片之间
- 智慧提醒卡片紧跟分割线下方

---

## 三、问候语

### 3.1 结构

| 元素 | 说明 |
|------|------|
| 大标题 | 时间段问好 + 用户姓名，如「下午好，何宇翔」，基本固定，极少变化 |
| 副标题 | 动态内容，根据时段、条件、上下文展示不同文案 |

### 3.2 副标题文案策略

副标题不是固定内容，需要维护一个文案库，结合当前时段、课程状态、天气、日期等条件动态选取。文案应自然、口语化，避免程序感。示例方向：

- 今天没课的周末下午 → 「今天没课，下午自由安排」
- 课程结束的傍晚 → 「今天的课都上完了」
- 有运动目标完成 → 「今天的目标达成了」
- 普通工作日早晨 → 「新的一天，加油」

> 文案库的质量直接决定这个功能是否自然，建议单独整理一份文案清单。

### 3.3 切换触发条件

副标题在以下任一条件满足时切换：

1. 用户手动刷新页面
2. App 从后台回到前台，且距上次展示超过 **30 分钟**
3. 跨越时段边界（如从上午进入下午）

### 3.4 切换动画

仅副标题参与动画，大标题不动。

| 阶段 | 行为 |
|------|------|
| 旧文字退出 | 向上位移 + 淡出，约 200ms |
| 新文字进入 | 从下方进入 + 淡入，约 200ms |
| 缓动曲线 | ease-in-out，非线性 |
| 总时长 | 约 350ms |

---

## 四、状态切换动画（问候语 ↔ 提醒卡片）

### 4.1 无提醒 → 有提醒

1. 问候语整体（大标题 + 副标题作为一个块）从**居中位置**向左上角位移，同时执行 scale 缩小
2. 动画进行至问候语**接近目标位置**时，分割线淡入出现
3. 分割线出现后，智慧提醒卡片从下方滑入 + 淡入

| 参数 | 值 |
|------|----|
| 问候语位移+缩放时长 | 约 350ms |
| 分割线出现时机 | 问候语动画进行至约 70% 时 |
| 卡片进入时长 | 约 250ms |
| 整体动画总时长 | 约 500ms |
| 缓动曲线 | ease-in-out |

**技术说明：** 动画过程使用 `scale` + `translate` 实现视觉效果；动画结束瞬间切换为真实的小字号布局（修改字号与间距数值），用户感知不到这次切换。这样可以保证最终静止状态的布局关系正确，不受 scale 变换影响占位。

### 4.2 有提醒 → 无提醒

以上动画的完全逆序执行：卡片先退出，分割线淡出，问候语整体从左上角回到居中并放大。

## 五、智慧提醒卡片

### 5.1 设计原则

- 首页同时只存在**一张**智慧提醒卡片
- 卡片内容根据当前时段和条件动态决定
- 无需展示的时段，卡片不出现，首页保持干净

### 5.2 场景与内容规则

#### 场景 1：上课日早晨（约 7:00–8:00）

**触发条件：** 当天有课，早晨时段打开

**卡片内容：**
- 主信息：今日课程提醒，说明上午有哪几门课
- 附加提示：带书提醒，点明具体科目，如「别忘了带虚拟仪器和模式识别的书」
- 底部附件：课程表卡片（可展开或直接跳转）

**文案示例：**
> 今天上午有虚拟仪器和模式识别两门课，别带错书了！

---

#### 场景 2：上课前 10–15 分钟

**触发条件：** 距下一节课开始时间在 10–15 分钟以内，学生大概率已在路上

**卡片内容：**
- 下一节课的课程名称
- 上课地点（教室）
- 距上课还有几分钟（倒计时）

**文案示例：**
> 模式识别  ·  教学楼 A302  
> 还有 12 分钟上课

---

#### 场景 3：上课中

**触发条件：** 当前时间在某节课的上课时间范围内

**卡片内容：**
- 仅显示一句话：「专心上课，下课见」
- 卡片样式应比其他场景更轻、更安静，不抢眼

> 设计意图：学生正在上课，app 应主动退后，不展示多余信息，不吸引注意力。

---

#### 场景 4：下课前 5 分钟，且后续有课

**触发条件：** 当前节课还剩约 5 分钟，且今日还有后续课程

**卡片内容：**
- 下一节课的课程名称
- 教室位置
- 距下节课开始的时间

**文案示例：**
> 下一节  高等数学  ·  教学楼 B201  
> 还有 35 分钟

---

#### 场景 5：最后一节课结束

**触发条件：** 今日最后一节课已结束

**卡片行为：** 不显示卡片，首页回归干净状态，交由问候语接管（问候语回到居中大字版本）。

---

#### 场景 6：中午上课前 30 分钟

**触发条件：** 下午第一节课开始前约 30 分钟

**卡片内容：**
- 下午课程概览
- 换书提示，说明需要带哪些书

**文案示例：**
> 下午还有两节课，记得把书换一下——线性代数和大学物理。

---

#### 场景 7：晚间查寝未完成（约 22:00 后）

**触发条件：** 当日查寝签到未完成，时间在 22:00 左右

**卡片内容：**
- 查寝提醒文案
- 查寝操作入口（卡片内嵌按钮或跳转）

**文案示例：**
> 今天还没有完成查寝签到，记得处理一下。

---

### 5.3 场景优先级

当多个场景条件同时满足时，按以下优先级取最高项展示：

1. 上课前 10–15 分钟（紧急提醒优先）
2. 下课前 5 分钟且有后续课（次级紧急）
3. 上课中（安静状态）
4. 晚间查寝未完成
5. 中午换书提醒
6. 早晨带书提醒
7. 无提醒（首页空状态）

## 六、底部导航栏

### 6.1 现状问题

当前底栏图标 + 文字标签的形式有明显 Material3 风格，与整体设计语言不符。

### 6.2 调整方向

- 去除文字标签，仅保留图标
- 背景透明或极浅色，与页面自然融合
- 用一条细分隔线与内容区划分，不使用色块背景
- 图标选用语义明确的线性图标，三个页面（首页、课程表、我的）图标含义无需文字辅助即可识别

### 6.3 暂缓事项

当前底栏结构（三个一级页面使用底栏导航）暂维持不变，视觉风格调整优先级高于结构调整。
