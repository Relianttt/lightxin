# 轻小信 - 正式版预期功能补充

## 说明

本文档用于补充 `项目规划.md`，仅描述正式版阶段的增强方向，不改变当前 MVP 的开发边界。

当前阶段仍以 MVP 为主，优先保证以下主链稳定可用：

- 登录
- 课程表
- 跑步首页
- 真实跑步上传
- 模拟提交上传
- 劳动教育总览与详情

以下能力暂不纳入当前 MVP 交付范围，统一放入正式版规划。

## 正式版增强方向

### 1. 设置中心

新增独立设置页，用于承载正式版的个性化配置与高级功能开关，包括但不限于：

- 跑步相关设置
- 轨迹与路线管理
- 数据展示偏好
- 通知与权限状态检查
- 实验性功能开关

### 2. 自定义模拟路线

在设置页中新增 `自定义模拟路线` 模块，允许用户基于本人真实校园跑步习惯建立路线库。

目标能力：

- 手动录制个人校园路线
- 保存为本地路线模板
- 查看路线名称、总长度、录制时间
- 删除或重命名已保存路线
- 设置默认模拟路线
- 模拟提交时优先使用个人路线模板生成轨迹

### 3. 真实路线录制与复用

为保证模拟提交的轨迹更贴近真实使用习惯，正式版计划支持将真实跑步过程中的轨迹沉淀为可复用模板。

预期方案：

- 用户在校园内完成一次真实跑步后，可选择“保存为路线模板”
- 路线模板保存轨迹点序列、长度、时间分布特征
- 后续模拟模式可基于该路线做截取、插值、循环拼接
- 可选保留轻微随机偏移，使轨迹不完全重复

### 4. 模拟轨迹生成升级

当前 MVP 的模拟轨迹以“预设校园闭环点 + 均匀时间分布 + 轻微抖动”为主，满足可用性，但不追求高保真。

正式版将升级为更接近真实跑步路线的生成方式：

- 使用更长的路线 polyline，而不是少量闭环点重复循环
- 按线段长度进行轨迹插值
- 根据目标距离截取或拼接路线
- 使速度变化、拐点分布、点间距更接近真实运动轨迹

### 5. 路线质量校验

为了避免低质量录制路线影响后续使用，正式版计划增加路线模板质量检查。

例如：

- 轨迹点数量过少时提示重录
- 总长度过短时禁止保存为模板
- 漂移过大时提示该路线不适合用于模拟
- 路线超出校园范围时标记为异常

### 6. 查寝任务状态精细化

当前 MVP 阶段，查寝任务列表优先保证“可查看、可进入、可提交”的主链稳定，任务状态展示仍偏简化。

正式版计划对查寝任务状态做更细粒度的判定与展示，避免将“未到签到时间”的任务直接显示为“未签到”。

预期能力：

- 区分 `未开始 / 可签到 / 已签到 / 已结束 / 超出范围`
- 按接口真实时间窗判断任务是否进入可签到阶段
- 结合签到结果、定位范围、照片要求等条件展示更准确状态
- 列表页与详情页保持一致的状态口径

实现原则：

- 以真实接口字段为准，不用前端猜测时间状态
- 参考原 APP 的最终状态判定逻辑
- 必要时补抓列表接口与详情接口中的时间限制字段、状态字段与边界分支

## 与 MVP 的边界

以下内容明确不纳入当前 MVP：

- 设置页完整实现
- 自定义路线录制
- 路线模板管理
- 基于真实路线的高保真模拟算法
- 路线质量分析与评分
- 查寝任务状态精细化重构

## 结论

正式版将把“设置页 + 自定义模拟路线 + 真实路线复用”作为跑步模块的重要增强方向。

当前 MVP 继续保持聚焦：

- 优先保证现有接口链路稳定
- 优先保证首页、真实跑步、模拟提交、结果反馈可正常工作
- 暂不引入高复杂度的路线管理系统
