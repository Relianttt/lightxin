# 轻小信 (LightXin) - 项目规划

## Context

基于对「爱信校园」(cn.edu.aiit.axx) APP的逆向分析，已完整梳理出15+功能模块的接口、加密方式和认证机制。现在要基于这些已知接口，开发一款轻量替代应用「轻小信」，使用 Kotlin + Jetpack Compose，全原生实现，追求 Anthropic/iOS 18 风格的精致UI。

## 决策汇总

| 决策项 | 选择 |
|--------|------|
| MVP范围 | 登录 + 课程表 + 查寝签到 + 跑步运动 + 劳动教育 |
| 定位方案 | Android原生 LocationManager，无地图SDK |
| 跑步UI | 纯数据仪表盘，后期可扩展轨迹 |
| 跑步模式 | 双模式：真实GPS跑步 + 数据模拟提交 |
| 原生vs H5 | 全部 Compose 原生重写 |
| 架构 | Clean Architecture 三层 (data/domain/ui) |
| 项目组织 | 单Gradle模块 + 功能包分离 |
| 设计风格 | Anthropic / iOS 18，精致美观 |

## 技术栈

| 类别 | 选择 | 理由 |
|------|------|------|
| 语言 | Kotlin 2.0+ | 官方推荐，Compose原生 |
| UI | Jetpack Compose (BOM latest) | 声明式UI，AI友好 |
| DI | Hilt | Android标准，Compose集成好 |
| 网络 | Retrofit + OkHttp | 原APP用OkHttp，接口对齐 |
| 本地存储 | DataStore | 轻量，替代SharedPreferences |
| 导航 | Compose Navigation | 官方方案 |
| 状态 | ViewModel + StateFlow | 标准Compose模式 |
| 图片 | Coil | Compose原生支持 |
| 相机 | 原生相机 Intent | 查寝拍照用，调用系统相机APP |
| Min SDK | 26 (Android 8.0) | Compose最低实用版本 |
| Target SDK | 35 | 最新 |

## 核心基础设施详细设计

### 1. 多域名网络层

原应用涉及多个服务器域名，需要为不同域名提供不同的Retrofit实例：

```
ApiConstants.kt:
  BASE_MAIN    = "https://in.aiit.edu.cn"          # 认证+主服务
  BASE_CSH     = "https://in.aiit.edu.cn/zhxy-csh"  # 课表
  BASE_CHECKIN = "https://fdygl.aiit.edu.cn"        # 查寝
  BASE_SPORTS  = "http://sports.aiit.edu.cn:8082"   # 运动
  BASE_LABOR   = "https://ldjy.aiit.edu.cn"         # 劳动教育
```

通过 Hilt 的 `@Qualifier` 注解区分不同的 Retrofit 实例。共享同一个 OkHttpClient（含AuthInterceptor）。

### 2. 通用认证参数注入

`AuthInterceptor` 自动为请求注入通用参数：
- `access_token`
- `_userCode` / `userId`
- `_userName` (URL编码)
- `_userType`
- `appId = "11111111111111111111111111111111"`

不同域名的认证方式有差异：
- 主站接口：通用认证参数（Form表单追加）
- 查寝接口：JSON body + Bearer token header
- 运动接口：`studentCode` header + WebView参数
- 劳动教育：`userCode` + `accessToken` 表单参数

`AuthInterceptor` 需根据请求URL判断注入方式。

### 3. RSA加密工具

两套公钥，两个加密方法：
- `RSAUtils.encryptPassword(plain)` → 用 publicKey，登录密码加密
- `RSAUtils.encryptSportData(plain)` → 用 publicKey2，跑步数据加密

算法: `RSA/None/PKCS1Padding`，分段加密，Base64编码。

### 4. WGS-84 → BD-09 坐标转换

原生LocationManager返回WGS-84坐标，服务端期望百度BD-09坐标。
转换链路: WGS-84 → GCJ-02 (国测局) → BD-09 (百度)
这是一个纯数学变换，约20行代码，无需任何SDK依赖。

### 5. Design System (Anthropic/iOS 18 风格)

**设计语言核心特征**:
- 大圆角卡片 (16-24dp radius)
- 柔和阴影 + 微妙层次感
- 粗体标题 + 清晰信息层级
- 留白充足，呼吸感强
- 色彩克制：主体灰白调，点缀色突出操作
- Material3 主题，固定色板（不使用Dynamic Color）
- 深色模式

**色板方向**:
- 背景: 暖灰白 (#FAFAF8)
- 卡片: 纯白 + 轻阴影
- 主色: 柔和蓝 (可参考 Anthropic 品牌色调)
- 强调色: 橙/珊瑚 (用于CTA按钮)
- 深色模式: 深灰底 (#1A1A1A) + 微亮卡片

## 各功能模块详细设计

### 模块1: 登录认证

**接口**:
- `POST /zhxy-information/mobile/getToken.do` — 登录
  - 参数: `userCode`(学号), `password`(RSA加密)
  - 返回: `access_token`, `refresh_token`, `userCode`, `userName`, `userType`, `fileAddress`
- `POST /zhxy-information/mobile/refresh.do` — Token刷新

**页面**:
- LoginScreen: 学号输入 + 密码输入 + 登录按钮
- 登录成功 → 存储Token → 跳转Home

**要点**:
- 密码用 `RSAUtils.encryptPassword()` 加密
- Token/UserInfo 存入 DataStore
- 支持记住学号（本地存储）

### 模块2: 课程表

**接口**:
- `POST /zhxy-csh/dict/selectZcList.do` — 获取周次列表 + 当前周
  - 返回: `rows`(第0周~第17周), `data.week`(当前周), `data.schoolYear`, `data.schoolTerm`
- `POST /zhxy-csh/mobile/selectStuSelfTimeTable.do` — 获取指定周课表
  - 参数: `userCode`, `xn`(学年), `xq`(学期), `zc`(周次)
  - 返回: `rows[星期].kcVoList[节次]`，每课含 `kcmc/ksjc/jsjc/jsmc` 等

**页面**:
- ScheduleScreen: 7列网格，横轴=星期，纵轴=节次
- 周次选择器（顶部横滑）
- 当天高亮
- 课程卡片点击展开详情（教室、教师）

### 模块3: 查寝签到

**接口** (fdygl.aiit.edu.cn):
- `POST /app/dorm/pageStudentSignIn` — 查询签到任务列表
  - JSON body: `{pageNum, pageSize, type:"1", taskMajorType:"3"}`
- `POST /app/dorm/getTaskInfoByDateId` — 任务详情
  - JSON body: `{dateId: "..."}`
  - 返回: 签到条件、位置范围、是否需照片
- `POST /app/file/uploadFileToFastdfs` — 上传照片
  - Multipart文件上传
- `POST /app/dorm/signIn` — 提交签到
  - JSON body: `{signinPhoto, signinPlace, taskDateId, outSignin:"0", latLng:"lng,lat", outSigninDesc:""}`

**页面**:
- CheckinListScreen: 任务列表（未签到/已签到状态）
- CheckinDetailScreen: 签到操作页
  - 显示签到要求（位置范围、是否需要照片）
  - 相机拍照区域
  - 当前定位状态
  - 一键签到按钮

**要点**:
- 定位获取后需 WGS-84 → BD-09 转换
- `latLng` 格式为 `"经度,纬度"`（注意不是纬度,经度）
- 照片上传到 `fdygl.aiit.edu.cn/app/file/uploadFileToFastdfs`，拿到URL后再提交签到
- 查寝接口认证方式与主站不同，走独立的token/header机制

### 模块4: 跑步运动（双模式）

#### 4A. 运动首页

**接口**:
- `POST /mobile/sportApp/startIndex.do` — 首页统计数据
  - 返回: `todayMile`, `dsTaskMile`, `dsTaskThreshold`, `totalTaskThreshold` 等
- `POST /mobile/sportApp/checkDsStudent.do` — 学生类型判定

**页面**:
- RunningHomeScreen: 今日/总里程统计 + 任务进度 + 开始跑步/模拟模式切换

#### 4B. 真实GPS模式

**流程**:
1. 调用 `POST /mobile/sportApp/startRunning.do` 获取 `extraId`
2. 将 `extraId` 映射为 `exerciseId`
3. 启动前台Service `RunTrackingService`，持续GPS定位
4. 实时计算距离（Haversine公式）、配速、时长
5. 仪表盘实时更新：距离(km)、时长、配速(min/km)、速度(km/h)
6. 结束时:
   - `GET /mobile/time/getServerTime.do` 获取服务器时间作为endDate
   - 构造JSON: exerciseId, memberId, runningType:"1", startDate, endDate, mile, time, speed, runningRoute, sourceInfo
   - 全字段RSA加密 (publicKey2)
   - `POST /mobile/extra/addExtraCheckNew.do` 上传 (header: studentCode)

**RunTrackingService**:
- 前台通知显示跑步状态
- GPS间隔: 3-5秒
- 距离计算: Haversine公式 (替代百度DistanceUtil)
- 坐标收集: WGS-84实时坐标，上传前批量转BD-09

#### 4C. 模拟提交模式

**页面** RunningSimScreen:
- 距离输入 (km)
- 时长输入 (分钟) 或自动计算（基于合理配速）
- 开始时间选择（默认当前时间前推N分钟）

**轨迹生成算法** (TrajectoryGenerator):
1. 预设校园内的若干路线模板（坐标点序列，BD-09）
2. 根据目标距离截取/循环路线
3. 沿路线均匀分布轨迹点，间距约30-50米
4. 添加微小随机偏移（模拟真实GPS漂移，±2-5米）
5. 确保数学一致性: speed = mile / time * 3600
6. 速度限制在合理范围 (6-15 km/h 慢跑区间)

**上传流程**: 与真实模式相同的加密+上传链路。

#### 4D. 跑步数据加密细节

上传时的加密流程:
```
原始JSON → 逐字段取出 → RSA publicKey2 加密字段名和值 
→ 组装新JSON → 放入JSONArray → toString → 作为表单字段"list"提交
```

注意: 字段名和字段值都要加密，不是只加密值。

### 模块5: 劳动教育

**接口** (ldjy.aiit.edu.cn):
- `POST /anXiaoXin/queryPersonalTimesTotal.do` — 工时总览
  - 返回: `voluntaryTimes`(志愿), `summerTimes`(暑期), `laborTimes`(劳动), `socialTimes`(社区), `otherTimes`(其他)
- `POST /anXiaoXin/queryTimesDetailsPage.do` — 活动记录列表
  - 参数: `pageNo`, `pageSize`
  - 返回: 分页活动列表，每条含 `id`, `projectTypeName`, `activityName`, `serviceTimes`, `createDate`
- `POST /anXiaoXin/queryTimesDetails.do` — 单条详情
  - 参数: `id`, `type`
  - 返回: `organizer`, `serviceTimes`, `activityName`, `activityType`, `activityLevel`, `createDate`

**页面**:
- LaborSummaryScreen: 工时总览卡片(5类工时环形图/条形图) + 活动记录列表
- LaborDetailScreen: 活动详情页

**要点**:
- 纯只读模块，无提交操作
- 认证参数: `userCode` + `xh` + `accessToken` 等表单参数
